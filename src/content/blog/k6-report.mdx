# K6 報告產生方式

## 目的

---

- 透過 K6 產生壓力測試數據
- 測試數據能後介面化閱讀

## 使用套件

---

- **壓測工具：** K6
- **時間資料庫：** InfluxDB
- **Dashboard：** Grafana

## 使用方式

---

> 透過 `dockerfile` 一次建立 `K6` 、 `Influx` 、 `Grafana` 的容器環境，並透過 `enviroment` 設定各服務間的關連。

- 範例程式碼：

  ```yaml
  version: '3.8'

  networks:
    k6:
    grafana:

  services:
    influxdb:
      image: influxdb:1.8
      networks:
        - k6
        - grafana
      ports:
        - '8086:8086'
      environment:
        - INFLUXDB_DB=k6

    grafana:
      image: grafana/grafana:8.5.21
      networks:
        - grafana
      ports:
        - '3000:3000'
      environment:
        - GF_AUTH_ANONYMOUS_ORG_ROLE=Admin
        - GF_AUTH_ANONYMOUS_ENABLED=true
        - GF_AUTH_BASIC_ENABLED=false
        - GF_SERVER_SERVE_FROM_SUB_PATH=true
      volumes:
        - ./dashboards:/var/lib/grafana/dashboards
        - ./grafana-dashboard.yaml:/etc/grafana/provisioning/dashboards/dashboard.yaml
        - ./grafana-datasource.yaml:/etc/grafana/provisioning/datasources/datasource.yaml

    chronograf:
      image: chronograf:1.8
      ports:
        - '8888:8888'
      environment:
        - INFLUXDB_URL=http://influxdb:8086
        - INFLUXDB_USERNAME=root
        - INFLUXDB_PASSWORD=root
        - INFLUXDB_SKIP_VERIFY=true
      depends_on:
        - influxdb
      networks:
        - k6

    k6:
      image: grafana/k6:0.46.0
      networks:
        - k6
      ports:
        - '6565:6565'
      environment:
        - K6_OUT=influxdb=http://influxdb:8086/k6
      volumes:
        - ./scripts:/scripts
  ```

## 儀表版設定檔

> 為了要快速建立 `Grafana` 的樣式，統一使用 `grafana-dashboard.yaml` ﹑`grafana-datasource.yaml`、`dashboard/k6-default-dashboard.json` 進行設定

### Grafana-dashboard.yaml

> 這個檔案控制 dashboard 基本設定

```yaml
apiVersion: 1
providers:
  - name: 'testFrank'
    org_id: 1
    folder: ''
    type: 'file'
    options:
      path: /var/lib/grafana/dashboards
```

- **apiVersion：** 指定 Grafana 的 API 版本。
- **providers：** 定義 dashboard 的相關設定
- **options：** 定義 dashboard 樣式路徑（docker 中的路徑）

### Grafana-datasource.yaml

> 這個檔案控制 Grafana 連接 Influx 時的設定

```yaml
apiVersion: 1

datasources:
  - name: k6influxdb
    type: influxdb
    access: proxy
    database: k6
    url: http://influxdb:8086
    isDefault: true
```

- **name：** 定義 Data Sources 的名稱
- **type：** 定義要連接的資料庫類型
- **access：** 定義訪問資料庫的方式

  - proxy：Grafana 將作為中介，從 Grafana 伺服器向數據源發送查詢請求，然後再將結果返回到用戶的瀏覽器。這種方式適用於防火牆限制和安全性要求較高的情況。
  - direct：用戶的瀏覽器將直接向數據源發送查詢請求，並接收數據源返回的結果。這種方式通常用於公共可訪問的數據源，無需經過 Grafana 伺服器。
  - browser：Grafana 將通過 JavaScript 代碼在用戶的瀏覽器中向數據源發送查詢請求，然後在瀏覽器中處理數據源返回的結果。這種方式適用於一些需要前端交互的情況，例如使用 JavaScript 操作數據或將數據嵌入到其他應用中。

- **database：** 要連接的 `Influx` 資料庫名稱
- **url：** `Influx` 資料庫路徑
- **isDefault：** 將此資料庫設定為預設資料庫，

## 名詞解釋

---

<details>
	<summary>
		<b>version：</b>
	</summary>
	- 設定 `docker-compose` 要使用的版本，需要搭配 `docker` 的版本號決定。
</details>

<details>
	<summary>
		<b>networks：</b>
	</summary>
	  - **基本概念：** 可以把每個服務想像成一個家庭成員，每個服務運行在自己的 `房間`，也就是容器中。為了讓這些家庭成員能夠互相交流，我們可以定義一個 `客廳`，也就是 `networks`，通過這個客廳，這些家庭成員（容器）可以在同一個網路中互相通信。同時，也可以在每個服務的定義中指定它們能使用的網路，也就是定義誰可以進入這個 `客廳`。這樣，您可以根據需要控制容器之間的通信，確保只有特定的容器可以在同一個網路中交流。

    - **全域設定：** 透過全域設定 `network` ，定義各容器的溝通管道。

  </details>

<details>
    <summary>
    	<b>services：</b>
    </summary>

    - **基本概念：** `service` 是 Docker Compose 文件中的一個關鍵字，用於定義要運行的容器。每個 `service` 定義將對應到一個 Docker 容器的運行配置。
    - **全域設定：** 透過全域設定，定義了這份 docker-compose 文件會產生 3 個容器服務。

  </details>

    <details>
    <summary>
    	<b>image：</b>
    </summary>

    - 設定該服務要使用的 docker 要使用的映像檔。

  </details>

        <details>
    <summary>
    	<b>ports：</b>
    </summary>

    - **基本概念：** 設定 `ports` 可以讓本機訪問服務。

  </details>

        <details>
    <summary>
    	<b>enviroment：</b>
    </summary>

    - **基本概念：** `environment` 是 Docker Compose 文件中 `service` 部分的一個屬性，用於定義容器運行時所需的環境變數。這些環境變數可以影響容器內應用程序的行為，例如配置文件、資料庫連接等。

  </details>

    	    <details>
    <summary>
    	<b>volumes：</b>
    </summary>

    - **基本概念：** `volumes` 配置允許您在主機和容器之間建立雙向的資料綁定。這意味著當您在主機上的檔案進行更改時，容器內的檔案也會同步更改，反之亦然。

    - 這對於開發和測試非常有用，因為您可以在本機上進行編輯，並立即在容器內查看更改後的結果，無需不斷地建立映像或重新啟動容器。這種雙向資料綁定是 Docker 中的一個強大功能，使開發和測試過程更加高效。

  </details>

## 額外補充

---

<details>
<summary>dockerfile v.s. docker-compose.yml</summary>

    - **Dockerfile：**

      `Dockerfile` 是用於創建 Docker 映像（Image）的腳本文件。它包含一系列指令，這些指令描述了如何在基於現有映像的基礎上構建您的映像。您可以在 `Dockerfile` 中指定基本映像、安裝軟體、複製檔案、設定環境變數等。當您運行 `docker build` 命令時，Docker 將根據 `Dockerfile` 的指令創建一個新的映像。`Dockerfile` 是用於定義映像的建構步驟和配置。

    - **docker-compose.yml：**

      `docker-compose.yml` 是用於定義和管理 Docker 應用程序的配置文件。它允許您定義多個容器、網路、卷以及它們之間的關係。通過 `docker-compose.yml`，您可以一次性配置和啟動多個相關聯的容器，並定義它們之間的通信和互動方式。這對於需要多個容器協同工作的場景非常有用，比如在開發環境中配置完整的應用堆疊。

    - 簡而言之，`Dockerfile` 用於定義映像的建構過程，而 `docker-compose.yml` 用於定義多個容器和它們之間的關係，以構建出一個完整的應用程序堆疊。兩者的角色和功能是不同的，但它們都是在 Docker 環境中用於配置和管理的文件。

  </details>
